克制美学 是一种主张“少即是多”、强调节制、平衡与分寸感的设计与生活哲学。它主张通过做减法来剔除冗余装饰，将力量集中于精准的比例、天然的质感和深层的意境，从而展现一种内敛、高级且富有张力的优雅状态。本次的博客更新是里程级的，因为它支持了 Markdown 格式的博客，能让其用极其简单的格式来实现精致的效果。

## 前言：为什么我选这条路

我更看重两件事：**内容**和**耐久**。内容是主角。我希望你离开这个站时，记住的是某个项目、某句话，而不是"那个背景特效挺酷"。安静的外壳，是为了让里面的东西被听见。至于耐久，霓虹渐变、酷炫粒子是这两年的潮流，但潮流会过期，三年后再看多半尴尬；而一片暖灰、一个沙色的橘，是那种十年后回来看也不别扭的搭配。我宁愿要一个第二眼、第三眼依然耐看的站，而不是只赢在第一眼。

这就是克制美学：把能省的全省掉，好让该讲究的地方，毫不含糊。所以我极其欣赏这些真正耐看的设计比如Material Design，虽然谷歌公司都放弃了这套方案。但是我依然认为这是当今最规范且严谨的极简美学。

## 正篇：这次博客更新了什么

### 原本的博客

这个博客系统的诞生本身就是为了极高的自定义属性诞生的。从 C 语言的教程就可以看出，里里面穿插了大量的外部 **HTML** 引用，从而表现出漂亮的动画。我知道这很酷炫，但是这不论对我，还是 AI 都是一个巨大的挑战，因为本身我的博客就使用了极其复杂的流程来加载，同时又在博客文章上使用了大量的 **HTML**，这非常的繁琐。而且大部分文章并不需要这么高级的特性。

### 新时代的到来

所以，**Markdown** 出来了。从技术上说，它从原本的把整篇文章连同 HTML 标签一起塞进一个 JavaScript 模板字符串、再当成脚本用 `<script>` 标签注入页面，变成了现在的一个独立的纯 `.md` 文件，由浏览器在前端直接取来、交给一个解释器实时翻译成 HTML。底层看似更复杂了，多了一个解释器、一个解析库，甚至为此放弃了双击本地文件预览的能力，但编辑这一端却被大大简化了。我可以用一个简单的 Python 工具新建文章，再用 **Typora** 这类所见即所得的编辑器来写，而不必打开麻烦的 **Visual Studio Code**。同时为了保证高级特性，我依然保留了 **HTML** 写法，为下一次的复杂文章留好后路。

## 结语

写到这里我才发现，这次更新本身，就是一次克制美学的实践。

Markdown 这套语法，本身就是"少即是多"的样板：几个符号：#`、`*`、`-`——就够表达标题、强调和列表，没有一行多余的标签。我把那层繁琐的包装删掉，把省下来的力气，全留给真正重要的东西：文字本身。而那条特意保留的 **HTML** 后路，又保证了真到需要的时候，我依然能放开手脚做复杂的东西——克制从来不是做不到，而是默认不做、需要时才做。
